## What worked

- **Deep link → Class Invite** is the strongest parent onboarding path — clear class title, sport, and dual CTAs (Choose child / Add child).
- **Child profile picker** shows class context in heading; empty state is honest (“No child profiles yet”) with inline **+ Add child profile**.
- **No Marcus demo leakage** — fresh parent sees zero children, not Alex/Mia.
- **Legal consent gate** from invite inline path matches Profile path — same clear “Legal review in progress” copy (not silent).

## Friction (prioritized)

| P | Screen | Issue | Suggested change | Contract impact |
|---|--------|-------|------------------|-----------------|
| P1 | Add Child (invite inline) | **`age_category` null** after 18+ signup → “Adults only” blocks parent before consent (same as `parent-first-child`) | Persist age from signup metadata to `profiles.age_category` | Fails enrollment prerequisite; add signup row |
| P2 | Add Child inline | `returnToInvite` param passed from invite/picker but **AddChildProfileScreen ignores it** — post-create may not return to enroll flow | After create + consent, navigate back to picker/confirm with invite context | Enrollment contract “inline add → join” gap |
| P2 | Discovery | Profile **Family** still hidden (shared with `parent-first-child`) — invite link is the viable entry for zero-child parents | Fix `shouldShowFamilySection` OR document invite-first onboarding (H2) | Cross-ref family onboarding contract |
| P3 | End-to-end | Lawyer gate blocks enrollment (`GUARDIAN_LAWYER_COPY_APPROVED=false`) | Expected v1 — TestFlight note | Documented in contract |
| P3 | Invite UX | No coach name on Class Invite screen (only class + sport) | Add coach display name per contract “coach name” row | Minor contract clarity |
