## Friction (prioritized)

| P | Screen | Issue | Suggested change | Contract impact |
|---|--------|-------|------------------|-----------------|
| P0 | Login handoff from onboarding | Host journey blocked at authentication. Credentials can be entered, but submit affordance is unclear/non-actionable in this state, so user cannot reach create/roster flows. | Add explicit primary CTA button (`Continue`/`Log in`) with loading/error states and accessibility role `button`; require deterministic route to authenticated home on success. | `flow-invite-to-rally` and `flow-rally-session` should include signed-in handoff acceptance criteria. |
| P1 | Login UI clarity | "Log In" appears as header-style static text, making the action model ambiguous for users. | Separate title and CTA visually and semantically; avoid tappable-title pattern. | `flow-invite-to-rally` copy/UI section should define action hierarchy on auth screen. |
| P1 | Tier-2 coverage impact | Because auth failed, host-specific path (`create -> roster -> lock`) could not be exercised in this pass. | Treat as blocker; rerun host persona immediately after auth fix verification. | Queue cannot be considered high-confidence for host workflows until this is green. |
